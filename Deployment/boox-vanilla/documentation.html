Documentation on How We Make This Project

Users will input the characteristics of the property that they are looking for in the search bar, our recommendation algorithm will automatically generate outputs as the user types. The output is based on the similarity of the description of units, and the input of the user.

The dataset that we used to generate the descriptions can be found here.

The descriptions are only based on some primary features, such as bedrooms, bathrooms, beds and amenities. Thus, there are limitations on what the user can input to achieve acceptable results. Our recommendation system will show a list of units, by clicking on one of the outputs, it will show an extended detail of each unit. One of the outputs that is shown is what cluster the unit belongs to, this feature can make it easier for the user to see any similar units. We used the Kmeans method for our clustering.

The details of which can be found here.

Here's some detail of our team clustering:

We used other methods besides Kmeans for the clustering, but based on the validation test that we did, we found that Kmeans produced the best results.
